AryaOS Site Configuration

Site-wide settings in /etc/aryaos/aryaos-config.txt, inherited by every sensor gateway (per-service overrides live in each tool's own Cockpit page). Changes apply on Save & restart sensors.

TAK destination

Default feeds the local Charontak hub; Charontak forwards to Mesh SA and/or a TAK Server (see Charontak lanes). Direct examples: tls://takserver.example.com:8089, udp+wo://239.2.3.1:6969.

Device role

Choose which sensor pipelines run on this unit. The CoT core (Charontak, LINCOT, GPSTAK, gpsd) always runs; the role enables its sensor services at boot and stops the rest.

Onboarding hotspot password

When no known Wi-Fi is in range, AryaOS broadcasts its onboarding hotspot (comitup). By default it is open — set a WPA2 password (8–63 characters) to protect it. Applies after the next reboot (or comitup restart).

Radio control & EMCON

Turn the Wi-Fi onboarding hotspot off once a unit is fielded on ethernet, or put the box in EMCON (radio silence) — rfkill blocks Wi-Fi and Bluetooth for low-emissions (LPI/LPD) operation. Ethernet is unaffected, SDR receivers keep running, and EMCON persists across reboot.

Radios (RTL-SDR)

AryaOS selects dongles by EEPROM serial: stx:1090:0 for ADS-B 1090 MHz (readsb / dump1090-fa) and stx:978:0 for UAT (matches the UAT serial setting above). Writing a serial briefly stops SDR services; replug the dongle (or reboot) before the new serial is visible.

#DeviceSerialNew serial

Site-wide TAK TLS certificates

Upload once; every PyTAK gateway inherits PYTAK_TLS_CLIENT_CERT/KEY/CAFILE from the site config. PEM format (for .p12 bundles, convert first: openssl pkcs12 -in bundle.p12 -out client.pem -clcerts -nokeys / -nocerts -nodes).

TAK connection

Import an ATAK/iTAK connection data package or paste a one-time TAK Server enrollment URL. AryaOS installs the certs under /etc/aryaos/tls and enables Charontak forwarding to the TAK Server.

Backup & restore

      
Save the full device configuration — site settings, TAK certificates,
        Charontak lanes, per-gateway settings, network, and Node-RED — to a restorable archive.
        Use it to migrate to a replacement box or keep a golden config.

      

        
      

      

        
        
      

      
Note: a backup with secrets contains
        private keys and Wi-Fi passwords in the clear. Store it securely.

      

      

        
        
      

      
    


    

      
OS image backup

      
Pull down a copy of the exact AryaOS image this unit was
        built from — a cold backup you can copy off the box and re-flash later.
        The download lands under /var/lib/aryaos/image/ (about 1.5 GB);
        retrieve it over the network afterwards.

      

        
        
      

      

      
    


    

      
Reset & decommission

      
Destructive maintenance actions. The device reboots when they finish.
        Consider creating a backup first (above).


      
Factory reset

      
Restores AryaOS to factory defaults — clears site config, TAK certificates,
        Charontak lanes, and device identity, then re-runs first-boot setup. Keeps the OS, installed
        packages, and (by default) the current Wi-Fi so a remote box is not stranded.

      

        
      

      

        
        
      

      

      
Zeroize (secure sanitize)

      
Shreds and removes every key, credential, network secret, log,
        recorded track, and identity, overwrites free space, and reboots clean. For decommissioning or
        before the box leaves your control. The box stays usable.

      
Flash-media limit: wear-leveling means overwrite/TRIM is
        best-effort, not a guarantee. For a hard guarantee use full-disk encryption or destroy the media.
